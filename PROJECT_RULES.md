# ExpiryGO - Project Rules

## General

Always inspect the existing codebase before making changes.

Never assume the project is empty.

---

## Development Rules

- Never recreate the project.
- Never reinstall React or Vite.
- Never overwrite working code.
- Never delete existing features unless asked.
- Keep the application working after every change.
- Fix any bug before adding a new feature.
- Make one feature at a time.
- Explain what files will be modified before editing.
- After every completed feature, stop and wait for approval.

---

## Code Quality

- Keep components reusable.
- Avoid duplicated code.
- Keep files organized.
- Use meaningful variable names.
- Do not create unnecessary files.

---

## Testing

After every feature:

- Check for build errors.
- Check browser console.
- Verify functionality.
- Fix any issues before continuing.

---

## Git Workflow

After a successful feature:

git add .
git commit -m "Describe the feature"
git push

Never commit broken code.

---

## Future Direction

ExpiryGO should become an offline-first smart pantry assistant.

Future phases include:

- Barcode scanning
- OCR receipt scanning
- AI recipe suggestions
- Firebase sync
- Shared family inventory
- Shopping insights
- Food waste analytics
